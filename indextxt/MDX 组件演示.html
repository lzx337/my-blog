⚛️ MDX 组件演示 · 在 Markdown 中释放 React 能量

2026年5月24日 约 8 分钟 1.2k 次浏览
分类：前端开发|标签：MDXReact组件

MDX 是一种强大的格式，它允许你在 Markdown 文档中无缝地导入和使用 React 组件。本文将演示如何构建交互式组件、自定义块和动态内容，让你告别静态文档的限制。

✨ 什么是 MDX？

MDX = Markdown + JSX。你可以像写普通 Markdown 一样书写文本、标题、列表，同时直接嵌入 React 组件并传递 props。它被广泛应用于 Next.js、Gatsby、Docusaurus 等框架中，用于编写技术文档、博客甚至设计系统。

# 这是一个 MDX 文档

你可以使用普通 Markdown 语法。

<Button variant="primary">点击我</Button>

<Alert type="info">这是一条信息</Alert>

🔄 状态与交互组件

MDX 组件可以拥有自己的 state，实现计数器、开关等交互。下面是一个简单的计数器演示（模拟 React 行为）。

点击 + / - 改变数值（演示内部状态）

import { useState } from 'react';

export const Counter = () => {
  const [count, setCount] = useState(0);
  return (
    <div>
      <h3>{count}</h3>
      <button onClick={() => setCount(c => c+1)}>+</button>
    </div>
  );
};

📁 导入外部组件库

你可以从本地或 npm 包导入组件，然后在 MDX 文档中使用。例如：

import { Tabs, Tab } from 'my-ui-lib';

<Tabs>
  <Tab label="Markdown">支持标准语法</Tab>
  <Tab label="JSX">支持 React 组件</Tab>
</Tabs>
（模拟 Tabs 效果）你可以轻松构建带选项卡的文档区域。

⚙️ 如何开始使用 MDX？

  1. 安装 @mdx-js/loader 或使用框架内置支持（Next.js 13+ 默认支持）。
  2. 创建 .mdx 文件，并在 frontmatter 中定义元数据。
  3. 在文件中导入组件并使用 JSX 语法。
  4. 配置 Webpack 或 Vite 使其处理 MDX 文件。
// next.config.js
const withMDX = require('@next/mdx')();
module.exports = withMDX({
  pageExtensions: ['js', 'jsx', 'mdx'],
});

💡 提示： 本博客暂时未启用完整的 MDX 运行时，但所有演示的代码语法均可在真实 MDX 环境中直接使用。你可以在 MDX 官网 找到更多高级用法。

MDX 将内容与交互完美结合，让技术文档不再是死板的文字，而是可玩、可测试、可演示的活文档。希望本指南能帮助你开启 MDX 的精彩世界！
