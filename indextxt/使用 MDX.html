📘 使用 MDX：在 Markdown 中运行 React

2026年5月24日 约 9 分钟 946 次浏览
分类：前端开发|标签：MDXReact

MDX 是一种革命性的格式，它允许你在 Markdown 文档中直接导入并使用 React 组件。无论是技术文档、博客还是设计系统，MDX 都能让你轻松构建丰富、交互式的内容。本文将带你从零开始，掌握 MDX 的安装、配置和核心用法。

🎯 什么是 MDX？

MDX = Markdown + JSX。你可以像平常一样使用 # 标题、- 列表、**粗体** 等 Markdown 语法，同时还能在任意位置插入自定义的 React 组件。这些组件可以拥有状态、事件处理和样式，让文档变成真正的“可交互应用程序”。

# 这是一个 MDX 文档

普通的 Markdown 内容...

<Button onClick={() => alert('Hi!')}>点击我</Button>

<Chart data={salesData} />

📦 安装与配置（Next.js 示例）

MDX 在主流框架中都有良好的支持。以 Next.js 为例，从 Next.js 13 开始，官方已经内置了 MDX 支持。只需安装 @next/mdx 和相关依赖即可。

npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx

在项目根目录创建 mdx-components.tsx（或 .js）文件，用于自定义组件映射：

import type { MDXComponents } from 'mdx/types';
 
export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components,
    // 自定义 Markdown 元素的渲染组件
    h1: (props) => <h1 className="text-3xl font-bold" {...props} />,
    a: (props) => <a className="text-primary hover:underline" {...props} />,
  };
}

然后修改 next.config.js：

const withMDX = require('@next/mdx')();
module.exports = withMDX({
  pageExtensions: ['js', 'jsx', 'mdx', 'ts', 'tsx'],
});

现在你就可以在 /pages 或 /app 目录下创建 .mdx 文件并直接访问了。

🚀 编写第一个 MDX 文件

创建一个 hello.mdx 文件：

---
title: 欢迎使用 MDX
---

# 你好，MDX！

这是一个 **非常棒** 的格式。

<button className="btn btn-primary" onClick={() => alert('Hello MDX')}>
  点我试试
</button>

上面的按钮将会渲染为一个可交互的按钮，点击时触发弹窗。这就是 MDX 的魔力。

✨ 导入并使用自定义组件

你可以在 MDX 文件中导入任意组件，就像在 React 文件中一样：

import { Counter } from '../components/Counter';
import { Alert } from '../components/Alert';

# 交互式文档

<Counter initialValue={10} />

<Alert type="warning">请注意：MDX 文件支持所有 React 特性。</Alert>

点击 ± 改变数值（组件内部状态）

// components/Counter.jsx
import { useState } from 'react';
export const Counter = ({ initialValue = 0 }) => {
  const [count, setCount] = useState(initialValue);
  return (
    <div>
      <button onClick={() => setCount(c => c-1)}>-</button>
      <span>{count}</span>
      <button onClick={() => setCount(c => c+1)}>+</button>
    </div>
  );
};

🧩 内置组件映射与样式

MDX 允许你覆盖默认的 HTML 元素（如 h1、code、img）的渲染方式。在上述 mdx-components 文件中可以全局定义，也可以为特定文件单独指定。此外，你完全可以使用 Tailwind CSS 或任何 CSS 框架为 MDX 内容添加样式。

💡 提示： 如果你想让代码块自动支持语法高亮，可以结合 rehype-highlight 或 remark-prism 等插件。Next.js 中可以通过配置 remark 和 rehype 选项实现。

📐 在 MDX 中使用数学公式与图表

MDX 同样可以集成 LaTeX 或图表库。例如，使用 remark-math 和 rehype-katex 在 MDX 中渲染数学公式：

$$
  \int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$

也可以导入 ECharts 或 Recharts 组件来绘制交互式图表。这样你的技术文档就能兼具数据可视化的能力。

🌐 部署与最佳实践

  • 使用 前端框架 的官方 MDX 集成（Next.js、Gatsby、VitePress、Docusaurus 等），避免手动配置。
  • 将常见的 UI 组件（按钮、卡片、告警框、视频播放器等）封装成可复用的 MDX 组件库。
  • 为 MDX 文件支持 Frontmatter 元数据（如标题、日期、标签），方便静态站点生成。
  • 注意性能：大型交互组件使用 dynamic import 懒加载，避免影响首屏加载速度。
  • 对于仅展示静态内容的组件，可以完全使用 Markdown 语法，保留 MDX 的优势仅限于需要交互的地方。
