Git 基础教程：从入门到精通

周六 4月 18 2026 约 20 分钟 1 次浏览
分类： 教程 | 标签： Git GitHub 分支管理

Git 是目前世界上最流行的分布式版本控制系统，由 Linux 之父 Linus Torvalds 于 2005 年创建。无论你是个人开发者还是团队协作，Git 都是不可或缺的工具。本教程将从零开始，带你掌握 Git 的安装、配置、核心命令、分支管理以及 GitHub 协作流程，助你真正精通版本控制。

一、Git 简介与核心优势

分布式版本控制系统：每个开发者本地都有完整的仓库历史，不依赖中央服务器。
快照而非差异：Git 记录每次提交时的完整文件快照，效率极高。
本地操作：绝大部分操作在本地完成，速度快，支持离线工作。

二、安装 Git

Windows / macOS / Linux

# Windows：官网下载安装包 https://git-scm.com/download/win
# macOS：brew install git
# Ubuntu/Debian：sudo apt install git
# 验证安装
git --version

首次配置（必须）

git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
git config --global init.defaultBranch main

三、核心概念：工作区/暂存区/仓库

工作区 → git add → 暂存区 → git commit → 本地仓库 → git push → 远程仓库。文件状态包括：未跟踪、已修改、已暂存、已提交。

四、基础命令（高频）

命令说明
git init初始化仓库
git status查看状态
git add .添加所有更改到暂存区
git commit -m "message"提交到本地仓库
git log --oneline简洁历史
git diff查看工作区差异

五、远程仓库协作 (GitHub)

# 克隆远程仓库
git clone https://github.com/用户名/仓库.git
# 查看远程
git remote -v
# 推送
git push origin main
# 拉取
git pull origin main
# 仅获取
git fetch origin

六、分支管理（精通关键）

git branch feature-login          # 创建分支
git switch feature-login          # 切换分支
git checkout -b new-branch        # 创建并切换
git merge feature-login           # 合并分支
git branch -d feature-login       # 删除分支
处理冲突：手动编辑冲突文件 → git add → git commit。

七、撤销与恢复

git restore README.md            # 丢弃工作区修改
git restore --staged README.md   # 从暂存区撤出
git reset --soft HEAD~1          # 撤销提交但保留修改
git reset --hard HEAD~1          # ❌危险！彻底删除
git revert HEAD                  # 安全撤销（推荐）

八、实用进阶技巧

.gitignore 忽略文件：node_modules/ .env dist/
git stash：临时保存修改
git tag：打版本标签
git bisect：二分查找问题提交

git stash push -m "临时保存"
git stash pop
git tag v1.0.0 -m "发布版本"
git push origin v1.0.0

九、典型工作流 + SSH免密

# SSH 配置（免密推送）
ssh-keygen -t ed25519 -C "你的邮箱"
cat ~/.ssh/id_ed25519.pub   # 添加到 GitHub SSH keys
git remote set-url origin git@github.com:username/repo.git

# 日常开发流程
git checkout -b feat-xxx
git add . && git commit -m "feat: xxx"
git push -u origin feat-xxx
# 合并到 main
git checkout main && git pull origin main
git merge feat-xxx && git push origin main

十、总结与最佳实践

  • 提交信息遵循 feat: / fix: / docs: 约定式规范
  • 不要对公共分支执行 git push --force
  • 定期同步远程代码 git pull --rebase
  • 敏感信息绝不上传，使用 .gitignore 或环境变量

核心速记

git status + git add . + git commit -m "" + git push 是每天最高频的操作。分支让协作变得优雅，SSH 让推送更加安全。

Git 的学习曲线虽陡，但掌握后将成为你开发中最强大的盟友。更多内容查阅 Pro Git 官方文档 或交互式学习 Learn Git Branching。
