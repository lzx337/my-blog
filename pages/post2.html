Markdown 与 MDX 写作指南

周四 5月 21 2026 约 15 分钟 1 次浏览
分类： 教程 | 标签： Markdown MDX React 数学公式

Markdown 是一种轻量级标记语言，由 John Gruber 于 2004 年创建，广泛用于文档编写与博客创作。MDX 是 Markdown 的超集，允许在 Markdown 中直接嵌入 JSX 和 React 组件。本篇将全面介绍 Markdown 语法、MDX 扩展以及如何在技术博客中使用数学公式、代码高亮、自定义组件等高级功能。

Markdown 基础语法

标题、段落、列表、链接等是日常写作最常用的元素：

# 一级标题
## 二级标题
**加粗文本**  *斜体文本*  ~~删除线~~
- 无序列表项
1. 有序列表项
[链接文字](https://example.com)
![图片替代文本](图片url)

MDX：在 Markdown 中运行 React 组件

MDX 让你在编写文档时直接导入并使用组件，极大增强交互能力：

import { Alert } from '@/components/Alert'
# 我的文章
<Alert type="info">这是 React 组件！</Alert>

💡 扩展能力：MDX 支持在花括号 {} 中写 JavaScript 表达式，并支持 ESM 导入导出。

数学公式（LaTeX）

通过 remark-math 和 rehype-katex 插件，可以在 MDX 中渲染数学公式：

行内公式: $\sqrt{a^2 + b^2}$
块级公式:
$$
L = \frac{1}{2} \rho v^2 S C_L
$$

实际渲染效果（需引入 KaTeX CSS）：行内公式 √(a²+b²) ，块级公式清晰展示空气动力学升力公式。

代码高亮

使用 Prism.js 或 highlight.js 让代码块拥有美观高亮，例如 JavaScript 示例：

```javascript
function greet(name) {
  console.log(`Hello, ${name}!`);
}
```

在 Nextra 等框架中还可支持行高亮、行号：```js {2,4-6}。

自定义 React 组件

创建全局组件 mdx-components.tsx 即可在任何 MDX 文件中直接使用，无需导入：

export function Protip({ children }) {
  return <div className="bg-purple-100 p-4 rounded">💡 {children}</div>;
}

⚙️ 集成配置（Next.js + MDX）

npm install @next/mdx @mdx-js/loader
// next.config.mjs
import createMDX from '@next/mdx'
const withMDX = createMDX({ options: { remarkPlugins: [remarkMath], rehypePlugins: [rehypeKatex] } })
export default withMDX(nextConfig)

博客实战与 SEO 优化

  • 使用 frontmatter 导出元数据（标题、描述、日期）
  • 语义化 HTML 结构（h1-h6）利于 SEO
  • 服务端渲染代码高亮，减少客户端负担
  • 通过自定义组件输出 JSON-LD 结构化数据

推荐工具：Docusaurus、VitePress、Nextra 均原生支持 MDX，可快速搭建技术文档站。

常见问题与解决

问题解决方案
数学公式变量未定义错误正确配置 remark-math + recma-mdx-escape-missing-components
Prettier 破坏 MDX 语法添加 {/* prettier-ignore */}
React 组件导致多余 <p> 嵌套确保组件标签与内容间无换行

总结与资源推荐

掌握 Markdown 基础 + MDX 组件扩展，可以构建出兼具内容与交互的现代化博客。建议从基础语法开始，逐步探索 JSX 与组件生态。

官方文档： MDX 官网 · KaTeX · GFM 规范

将 Markdown 的写作愉悦与 React 的组件能力结合，让技术写作更富有表现力。希望你也能享受 MDX 带来的创作自由。
