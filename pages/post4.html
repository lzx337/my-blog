Vue.js 3 Composition API 实战

周四 5月 21 2026 约 10 分钟 284 次浏览
分类： Vue | 标签： Vue3 Composition API 组合式函数

Vue 3 的 Composition API 提供了一种全新的逻辑复用和代码组织方式，相比 Options API 更加灵活、可维护。本文通过一个完整的「用户信息管理与统计面板」实战项目，深入讲解 ref、reactive、computed、watch 等核心概念，并展示如何将其抽离为组合式函数（Composables），让代码更优雅。

为什么需要 Composition API？

在大型组件中，Options API 会将同一个逻辑的代码分散在 data、methods、computed、watch 中，导致可读性差。Composition API 允许按「功能点」组织代码，并轻松抽离复用。

ref 与 reactive：响应式基石

ref 用于包装基本类型或对象，访问时需 .value；reactive 只用于对象，可直接访问属性。

<script setup>
import { ref, reactive } from 'vue'

// ref 基本类型
const name = ref('张三')
const age = ref(28)

// reactive 对象
const user = reactive({
  name: '李四',
  age: 35,
  job: '前端工程师'
})
</script>

<template>
  <input v-model="name" />
  <input v-model="user.name" />
</template>

computed：派生状态

根据已有数据自动计算并缓存，依赖变化时重新计算。

const ageGroup = computed(() => {
  if (user.age < 30) return '青年'
  if (user.age < 60) return '中年'
  return '老年'
})

watch 与 watchEffect：响应式副作用

watch 精准监听特定数据源，watchEffect 自动收集依赖并立即执行。

watch(() => user.age, (newAge, oldAge) => {
  if (newAge >= 60) alert(`${user.name} 已到退休年龄`)
})

watchEffect(() => {
  console.log(`用户 ${user.name} 今年 ${user.age} 岁`)
})

实战案例：用户信息管理面板

实现编辑姓名/年龄/职业，实时计算年龄组，年龄超过60岁弹出退休提醒，并提供重置功能。

<script setup>
import { reactive, computed, watch } from 'vue'

const initialUser = { name: '王小明', age: 28, job: 'UI 设计师' }
const user = reactive({ ...initialUser })

const ageGroup = computed(() => {
  if (user.age < 30) return '青年'
  if (user.age < 60) return '中年'
  return '老年'
})

watch(() => user.age, (newAge, oldAge) => {
  if (newAge >= 60 && oldAge < 60) {
    alert(`${user.name} 已到退休年龄，请安排交接`)
  }
  localStorage.setItem('userAge', newAge)
})

function resetUser() {
  Object.assign(user, initialUser)
}
</script>

<template>
  <div>
    <input v-model="user.name" />
    <input type="number" v-model.number="user.age" />
    <input v-model="user.job" />
    <p>年龄组：{{ ageGroup }}</p>
    <button @click="resetUser">重置</button>
  </div>
</template>

进阶：抽离为组合式函数 (Composables)

将用户管理逻辑封装到 useUser.js，实现跨组件复用。

// composables/useUser.js
import { reactive, computed, watch } from 'vue'

export function useUser(initialData) {
  const user = reactive({ ...initialData })
  const ageGroup = computed(() => /* 同上 */)
  watch(() => user.age, /* 退休提醒 */)
  function reset(original) { Object.assign(user, original) }
  return { user, ageGroup, reset }
}

✅ 优势：逻辑复用、清晰分离、易于测试，可配合 Vue Devtools 完美调试。

核心 API 速查表

API作用使用场景
ref包装任意值，返回响应式对象基本类型 / 需要整体替换的对象
reactive代理对象，深度响应管理一组状态，不会整体替换
computed派生缓存值基于已有状态计算新值
watch监听特定源执行副作用异步操作、DOM 操作、存储等
watchEffect自动收集依赖并立即执行不需要旧值的副作用
toRefs保持解构后响应性从 reactive 解构返回

最佳实践与避坑

  • reactive 不要整体重赋值：需用 Object.assign 或改用 ref 包裹对象。
  • 解构 reactive 会丢失响应性：用 toRefs 包裹再解构。
  • watch 默认不深度监听：监听 reactive 对象属性需要 getter 函数，监听整个对象用 { deep: true }。
  • 避免过度使用 ref：一组相关状态建议用 reactive 提高可读性。

总结

Composition API 不仅仅是语法变化，更是一种编程范式的升级。它让逻辑复用变得优雅，代码结构更清晰。通过本文的实战案例，你已经掌握了 ref、reactive、computed、watch 的核心用法，并学会了如何封装组合式函数。接下来推荐继续学习 Vue Router 4 和 Pinia，配合 Composition API 打造现代化应用。
